PrivateClaw is a companion client for a self-hosted OpenClaw + PrivateClaw deployment. The app does not require an end-user account, subscription, or third-party social login. It pairs with a one-time encrypted invite generated by the OpenClaw host.

Recommended review path (no Telegram / Discord account required):

1. Start the relay from the repository root:
   docker compose up --build relay

   Default relay endpoint: http://127.0.0.1:8787
   Health check: http://127.0.0.1:8787/healthz

2. Install and enable the OpenClaw provider on the host machine:
   openclaw plugins install @privateclaw/privateclaw@latest
   openclaw plugins enable privateclaw
   openclaw config set plugins.entries.privateclaw.config.relayBaseUrl http://<host-or-lan-ip>:8787

   Important: if the review device is separate from the host machine, replace 127.0.0.1 with a LAN- or Internet-reachable hostname/IP that the iPhone can reach. The provider derives the required /ws/provider and /ws/app endpoints from relayBaseUrl automatically.

3. Start a one-time pairing session directly from the OpenClaw CLI:
   openclaw privateclaw pair

   Optional group mode:
   openclaw privateclaw pair --group

4. In the iOS app, tap "Scan QR code" and scan the QR shown by the CLI, or paste the privateclaw://connect?... invite URI manually.

5. Send a message in the app. The provider forwards the message into OpenClaw and returns the response through the encrypted relay.

Behavior notes:
- No plaintext conversation content is exposed to the relay. The relay only sees ciphertext plus delivery metadata.
- The standard review flow does not require a public chat account. The CLI pairing command above is the simplest way to test.
- The app supports one-to-one sessions by default. In group mode, multiple paired installs can join the same encrypted room, and the app shows participant join/leave notices.
- When a session is close to expiry, the app/provider surface /renew-session. In group sessions, /mute-bot and /unmute-bot are also available.

If Apple Review needs a simpler reproduction path, the same pairing flow can also be started from any existing OpenClaw chat surface by issuing /privateclaw or /privateclaw group, then scanning the returned QR code in the app.
