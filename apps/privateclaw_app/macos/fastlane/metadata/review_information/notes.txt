PrivateClaw for macOS is a companion desktop client for a deployed OpenClaw + PrivateClaw installation. The app does not require an end-user account, subscription, or third-party social login. It pairs with a one-time encrypted invite generated by the OpenClaw host.

Recommended review path (production-style, no local development setup required):

1. Use an already-running OpenClaw deployment where the published `@privateclaw/privateclaw` plugin is enabled.
   The default production relay is `https://relay.privateclaw.us`, so no local relay setup is required for the normal review flow.

2. From any existing OpenClaw chat surface connected to that deployment, send:
   `/privateclaw`

   Optional group mode:
   `/privateclaw group`

3. In the macOS app, either scan the QR returned by OpenClaw or paste the same one-time `privateclaw://connect?...` invite URI into the invite field.

4. Send a message in the app. The deployment forwards the message into OpenClaw and returns the encrypted response through the relay.

5. If the session is close to expiry during review, use the in-app renew control (or send `/renew-session`) to continue the same encrypted session.

Behavior notes:
- No plaintext conversation content is exposed to the relay. The relay only sees ciphertext plus delivery metadata.
- The production review flow does not require the reviewer to start Docker, run a local relay, or install/configure the provider manually.
- The app supports one-to-one sessions by default. In group mode, multiple paired installs can join the same encrypted room, and the app shows participant join/leave notices.
- The desktop build supports real-time local notifications while the app is running.

If Apple Review needs a live invite for the current build, we can provide a fresh one-time pairing session from an already-running production deployment.
